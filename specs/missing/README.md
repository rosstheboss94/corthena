# Missing Specification Changes

**Status:** Working record  
**Owner:** Project  
**Last updated:** 2026-07-13  
**Related:** [Specification index](../README.md)

This directory records behavior and public-contract changes that have not been
incorporated into the authoritative specifications. These notes expose the
divergence; they do not replace or override the authoritative specifications.

Create one descriptively named, kebab-case Markdown file per change. Update an
existing note when later work concerns the same change instead of creating a
duplicate.

Each note must use this structure:

```markdown
# Change title

**Status:** Missing from authoritative specifications
**Recorded:** YYYY-MM-DD
**Affected:** Relevant implementation paths, contracts, or specifications

## What changed

Describe the observable behavior or public-contract change.

## Why

Describe the reason for the change.
```

Keep notes concise and factual. Remove a note only when its change is reverted
or the user explicitly incorporates it into the authoritative specifications.
